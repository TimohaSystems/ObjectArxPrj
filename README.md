# ObjectArxPrj
Приложение добавляет в автокад две команды:
1.  STR_REPLACE
Для работы команды в той папке, где находится рабочий чертёж (файл .dwg) должен находиться текстовый документ, имя которого должно совпадать с именем рабочего чертежа. Т. е. если чертеж называется "Проект.dwg", то текстовик должен лежать в той же папке и называться "Проект.txt".
Структура текстового файла должна быть такая: нечётная строчка должна содержать заменяемую в чертеже строку, а следующая за ней чётная строка содержать необходимое значение:

	имя_объекта
	Исполнительная съёмка г-да по г. Волгограду, просп. Ленина 100
	протяженность_трубы
	500 м

Всю прелесть этой команды можно прочувствовать, если чертежи приходится редактировать массово. В таком случае с помощью таблицы Excel и Windows CMD можно в два счёта создать кучу текстовиков с нужным содержимым и приставить их к чертежам.

ВАЖНО!!!
Команда редактирует ВСЕ объекты текста (будь то однострочный или многострочный текст) в пространстве модели и в пространстве ВСЕХ листов.


2. MAP_REQUEST
Она нужна для вставки разграфки. Сначала нужно начертить линию (которая, к примеру, обозначает трубопровод на плане), затем ввести команду MAP_REQUEST, затем Автокад предложит выбрать линию, после чего в чертёж вставится квадрат, соответствующий архитектурному планшету, покрывающему эту линию.
В центре квадрата будет наименование растра.
В менеджере слоёв создадутся слои под наименование и линию растра.

При первом использовании программы выйдет диалоговое окно, запрашивающее местоположение файла базы данных. Это MapData.db (файл базы данных SQLite). Без него работать не будет.

ВАЖНО!!!
В данный момент команда охватывает ТОЛЬКО растры по г. Волгограду в системе координат МСК-34 зона 1.
Диалоговое окно открывается немного коряво, исправлю со следующим обновлением. Поэтому если после первого выбора файла базы данных система опять просит его указать, нужно отменить выполнение команды и ввести её заново.
